	q: What happens if users choose "Cancel"?
	a: It shows: "You've chosen: No". Line 6 of the codebase shows that the program checks whether the user clicks on Yes (returns YES_OPTION value), it will show "You've chosen: Yes" corresponding. If not, including clicking on No, Cancel, or Cancel, the returned value differs from YES_OPTION, which results in "You've chosen: No" without classifying it into the other values.


	q: How to customize the options to users, e.g. only two options: "Yes" and "No", OR "I do" and "I don't" (Suggestion: Use Javadocs or using Eclipse/Netbean IDE help).
	a: Replace 'option' initialisation with this snippet:
		String[] options = {"I do", "I don't"};
        int option = JOptionPane.showOptionDialog(null, "Change to first class ticket?", null, JOptionPane.YES_NO_OPTION, JOptionPane.QUESTION_MESSAGE, null, options, null);

	showOptionDialog method:: public static int showOptionDialog(Component parentComponent, Object message, String title, int optionType, int messageType, Icon icon, Object[] options, Object initialValue)
		throws HeadlessException

	int optionType = YES_NO_OPTION makes the dialog have 2 default options "Yes" and "No", return YES_OPTION and NO_OPTION values by order
	int messageType = QUESTION_MESSAGE, which defaults to showing a question mark icon on the left
	Object[] options = options is an object array, indicating the choices for the user; modify it for alternative options' texts

	Edge case: optionType = YES_NO_OPTION, options = object array with more than 2 members
		The dialog shows an option for each member (more than 2, of course). Each option will return an integer, starting from 0 from left to right (YES_OPTION, NO_OPTION, CANCEL_OPTION, etc. are int).
